# 朋友局德州扑克 Web App

一个面向熟人朋友娱乐的手机端德州扑克工具 Demo（本地静态版）。

## 项目定位
- 熟人娱乐：仅供朋友之间组织牌局练习与学习。
- 手机端优先：界面优先适配手机浏览器竖屏。
- 虚拟筹码：仅展示和记录虚拟筹码。
- 不涉及真实金钱：不支持任何现金相关能力。

## 合规边界（明确禁止）
- 不充值
- 不提现
- 不抽水
- 不公开匹配
- 不开放陌生人大厅

## 当前阶段目标（第一阶段）
- 完成本地可运行的静态 Demo。
- 不依赖服务器。
- 不实现真实多人在线同步。
- 不做登录注册与支付功能。

## 已实现内容（第一版）
- 免责声明弹窗：必须点击“我已阅读并同意”后进入页面。
- 两个长期房间：房间 A / 房间 B（每个房间固定 12 个座位）。
- 预留房间码字段（当前不做校验）。
- 玩家入座流程：输入昵称、选择空座位。
- 座位信息展示：座位编号、昵称、筹码、状态。
- 角色概念预留：管理员 / 房主 / 玩家。
- 盲注与初始筹码设置（房主面板，前端数据层演示）。
- 补码记录基础结构（总买入、补码、带出、净输赢字段）。
- 基础牌局发牌演示：
  - 自动洗牌
  - 每名参与者 2 张手牌
  - 5 张公共牌（翻牌/转牌/河牌）
  - 本地 Demo 默认可见所有手牌（便于验证）
- 行动倒计时 UI 预留（45 秒、最后 10 秒高亮提示）。
- “暂时离开 / 返回 / 房主剔除”按钮和数据结构预留。

## 后续阶段规划
- 手机端进一步优化（交互、可读性、触控体验）。
- 完整下注流程（过牌/跟注/加注/弃牌）。
- 筹码账本增强与回合结算。
- 自动胜负判断。
- All-in 与边池逻辑。
- 多人在线实时同步。

## 文件结构
- `index.html`：页面结构和主要 UI。
- `style.css`：移动端优先样式。
- `script.js`：Demo 交互逻辑与数据结构。
- `docs/requirements.md`：项目需求文档（第一版整理）。

## 本地运行方法
1. 下载或克隆本仓库到本地。
2. 直接用浏览器打开 `index.html`。
3. 建议使用手机浏览器，或在桌面浏览器打开开发者工具切换手机视图。

## 提示
- 当前版本是“可学习、可演示”的最小可运行版本。
- 所有筹码均为虚拟筹码，不代表真实金额。
